# DataInsight Agent PRD v0.1

## 1. 产品名称

DataInsight Agent

---

## 2. 产品定位

DataInsight Agent 是一个面向数据分析师的 AI 分析搭子（AI Analyst Workspace）。

它帮助分析师完成从原始数据到周报/月报分析初稿的全过程，减少重复性的数据整理、分析和报告撰写工作。

与传统 BI 工具不同，DataInsight Agent 的核心目标不是提供复杂的数据建模能力，而是帮助分析师更快发现问题、更快完成分析、更快产出报告。

---

## 3. 产品愿景

让分析师把时间花在思考业务问题上，而不是重复制作报表。

---

## 4. 目标用户

### 核心用户（Primary User）

1~3年经验的数据分析师。

具备：

* Excel
* SQL
* Tableau / Power BI

基础能力。

痛点：

* 每周重复制作周报
* 每月重复制作月报
* 重复编写分析结论
* 重复进行数据清洗和数据检查

---

### 次要用户（Secondary User）

业务运营人员。

具备：

* Excel 使用能力

不具备：

* SQL
* Python

能力。

希望通过上传数据快速获得分析结果和业务洞察。

---

## 5. 用户核心场景

### 场景1：周报

每周五需要提交销售周报。

用户上传本周数据后：

* Agent 自动识别字段
* Agent 自动分析变化趋势
* Agent 主动发现异常变化
* Agent 生成周报草稿

分析师审核并修改后导出。

---

### 场景2：月报

每月初需要提交月报。

用户上传本月数据后：

* Agent 自动关联多张表
* Agent 自动生成趋势分析
* Agent 自动生成 KPI 分析
* Agent 自动生成业务总结

分析师审核后导出。

---

## 6. 核心价值

### 节省时间

将原本需要数小时完成的周报/月报分析工作缩短至约1小时以内。

---

### 降低重复劳动

减少：

* 数据清洗
* KPI统计
* 图表制作
* 报告撰写

等重复工作。

---

### 保持人工决策

Agent 负责分析。

分析师负责确认。

任何可能影响业务结论的关键推断都必须经过用户确认。

---

## 7. 产品原则

### 原则1

Agent负责猜测，人类负责确认。

例如：

* 字段映射
* 表关联
* KPI定义

必须由用户确认。

---

### 原则2

Agent不是决策者。

Agent可以：

* 发现事实
* 提出可能原因
* 提出可能建议

但必须明确标注：

* AI生成
* 可能原因
* 需人工确认

---

### 原则3

Agent输出的是80%初稿。

最终报告仍由分析师审核和修改。

---

### 原则4

产品围绕项目工作。

不是围绕文件工作。

项目保存：

* 数据源
* 字段映射
* KPI定义
* 历史分析
* 历史报告

---

### 原则5

Agent采用项目级记忆。

记忆属于项目。

不属于所有用户共享。

---

## 8. MVP范围

V1支持：

### 项目管理

* 新建项目
* 打开项目
* 保存项目

---

### 数据接入

* Excel上传
* CSV上传
* 多表上传

---

### 数据理解

* 字段自动识别
* 表关联自动识别
* KPI自动识别

并允许用户确认。

---

### 数据质量

* 缺失值分析
* 重复值分析
* 异常值分析
* ID字段识别

---

### 探索分析

* 数值分析
* 类别分析
* 相关分析

---

### 业务分析

* 时间趋势
* Top N分析
* 环比
* 同比
* 维度对比

---

### Agent分析

* 主动发现异常变化
* 主动发现贡献变化
* 自动生成分析结论

---

### 报告中心

* Word导出
* PPT导出
* 周报生成
* 月报生成

---

## 9. MVP不做什么

V1不支持：

* 企业级部署
* 多人协作
* 数据仓库
* 在线PPT编辑
* 在线拖拽排版
* 预测模型
* Power BI替代能力
* Tableau替代能力

---

## 10. 一句话总结

DataInsight Agent 不是另一个 BI 工具。

它是一个帮助分析师完成周报和月报工作的 AI 分析搭子。
